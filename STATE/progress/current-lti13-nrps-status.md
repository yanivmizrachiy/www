# מצב אמת נוכחי — LTI 1.3 / NRPS (V1)

עודכן: 1/6/26
Teacher Release: **NO** (ללא שינוי)
מקור אמת עליון: `PROJECT_RULES.md`

מסמך זה מסכם את מצב האמת הנוכחי של חיבור LTI 1.3 ושל קריאת
המשתתפים (NRPS) במרחב הפעיל, נכון לסיום סדרת ה-PRs ‎#232–#239.
הוא מתעד מה הושלם בקוד ומה עדיין דורש אימות חי (live) מתוך Moodle.
מצב שלא אומת מסומן במפורש `לא אומת` ולא `עובד`.

  אימות בלתי-תלוי מתוך הרצה חיה בריפו זה — `לא אומת`.
- **216 תלמידים** סונכרנו נכון לפי הקשר חי שמסר המשתמש.
  אימות מתוך הרצה חיה בריפו זה — `לא אומת`.
- **222 משתתפים בסך הכול** לפי הקשר חי שמסר המשתמש.
  אימות מתוך הרצה חיה בריפו זה — `לא אומת`.
- כל המספרים האלה (216 / 222) נמסרו כהקשר חי מהמשתמש. אין מספרים
  קשיחים בקוד; הספירות בקוד מגיעות אך ורק מתשובת NRPS החיה.

## מצב מורים (תצוגה / סיווג)

- הקוד לתצוגה ולסיווג מורים מוזג (PRs ‎#232–#239 — ראה למטה).
- אימות חי של שמות המורים, מספר המורים המדויק, וסיווג התפקיד
  הנכון מתוך הרצה חיה ב-Moodle — **`לא אומת`** עד שבדיקות ה-PR
  החיות יושלמו. הקוד מוזג; האימות החי נותר `לא אומת`.

## מה מוזג בקוד בסדרה זו (#232–#239)

- **PR #232 — בוצע.** Dashboard: תצוגת מורים ופירוט משתתפים
  מסשן NRPS נוכחי (preview עם token).
- **PR #233 — בוצע.** עמוד תלמידים: תצוגת NRPS לסשן הנוכחי,
  רשימת לומדים בלבד, קישורי פרופיל תקפים בלבד.
- **PR #234 — בוצע.** API לפירוט משתתפים (counts-only) מהסשן
  הנוכחי.
- **PR #235 — בוצע.** סיווג תפקידים (role classification) רב-שימושי
  ועמיד ל-NRPS.
- **PR #236 — בוצע.** סנכרון NRPS בבעלות השרת (token-only); ה-payload
  של הלקוח הורד לרמת fallback תאימות בלבד ואינו אוטוריטטיבי.
- **PR #237 — בוצע.** עמוד `/teachers` (צוות הוראה) מתוך preview בטוח,
  עם קישור מכרטיס המורים ב-Dashboard.
- **PR #238 — בוצע.** קישור פרופיל תלמיד מתוך Supabase fallback +
  ריקון רגוע של טאבים ריקים.
- **PR #239 — בוצע.** עמוד אבחון בידוד מרחב בטוח (space isolation) חי.

## בדיקות חיות לכל PR / הבדיקות הבאות הנדרשות עכשיו

יש לפתוח את הכלי מתוך Moodle בהשקה (launch) אמיתית ולאמת חי:

- LTI 1.3: launch אמיתי שמחזיר claim של NRPS תקף לסשן הנוכחי. `לא אומת`
- ספירת לומדים: `learners_inserted + learners_updated` שווה למספר
  הלומדים החי, ושלומדים שכבר סונכרנו מתעדכנים במקום להשתכפל. `לא אומת`
- שה-fallback של payload הלקוח אינו בשימוש שקט בפרודקשן
  (הציפייה: `used_legacy_payload: false`). `לא אומת`
- ספירות מורים / לומדים / משתתפים (למשל 216 / 222) — מוצגות אך ורק
  מ-NRPS חי. `לא אומת`
- שמות מורים, מספר המורים המדויק, ושם הקורס/המרחב. `לא אומת`
- שהתאמת "זה אתה" נפתרת עבור המורה הפותח (לפי שם תצוגה בלבד). `לא אומת`
- שהתאמת שם הלומד החי לרשומת תלמיד מיובאת מאפשרת קישור פרופיל תקף;
  שמות שונים (כינויים/סדר) יוצגו ללא קישור ולא יקשרו לפרופיל שגוי. `לא אומת`
- בידוד רב-מורים / רב-קורסים — לא אומת חי. `לא אומת`

אינו פעיל בפרודקשן.
