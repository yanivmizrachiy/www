## ממצאי דוח מרחב Moodle אמיתי של משרד החינוך — השלכות לפרויקט ולמדריך

מקור: בדיקה ישירה במרחב `moodlemoe.lms.education.gov.il` (קורס לדוגמה 1453, תפקיד מורה) בשילוב התיעוד הרשמי של Moodle. הדוח המלא הוזן ליניב־זיכרון של הפרויקט בתאריך 2026-07-07.

### 1. מה הדוח מוכיח על Moodle של משרד החינוך

- ליבת Moodle קיימת בכל מרחב־לימוד של משרד החינוך, ללא תלות בבית ספר, מחוז או מקצוע.
- כיכולות בסיס בכל מרחב: **דוחות יומני מעקב (Logs), דוח השתתפות, דוח השלמות פעילות, ויומן הציונים (Gradebook)** — כולל ייצוא לאקסל/CSV/ODS.
- תוספי משרד החינוך מוסיפים שכבת אנליטיקה ואוטומציה מעל הליבה:
  - **ניהול למידה** — לוח מחוונים אנליטי (KPIs + 9 גרפים) עם ייצוא SVG/PNG/CSV לכל גרף.
  - **דוח מסכם לבגרות — סמל שאלון** — סיכום ציונים לפי סמל שאלון (זמין רק במרחבים שהוגדר בהם סמל שאלון).
  - **ניהול קבוצות בפעילויות** — הגדרת מצב קבוצות לכל הפעילויות בבת אחת.
  - **תזכורות** (Reminders) — שליחת תזכורות אוטומטיות במייל לפני מועדי הגשה ואירועים.
- הפלטפורמה תומכת ב-LTI (External Tool) ובמסגרת Web Services מלאה עם token-ים והרשאות ברמת מנהל.

### 2. מקורות אמת ל־Teacher Hub

כל מקור להלן קיים במרחב אמיתי של משרד החינוך, ולכן מותר להסתמך עליו כמקור נתונים אמיתי (בכפוף להרשאות ולזמינות token):

| מקור | מטרה עבור Teacher Hub |
|------|----------------------|
| משתתפים (Participants) | רשימת תלמידי הכיתה, שיוך לקבוצות, גישה אחרונה, סטטוס |
| ציונים / Gradebook | ציונים אמיתיים, פריטי ציון, משוב, ייצוא Excel/CSV/XML |
| דוחות | פעילות מרחב, השתתפות, ניהול למידה |
| יומני מעקב / Logs | מקור אמת ל-Practice Time ולזיהוי דפוסי כניסה |
| יומני זמן־אמת | פעילות חיה כרגע (אם זמין למורה) |
| השלמות פעילות (Activity Completion) | מטריצת תלמיד־מול־פעילות עם סימוני השלמה |
| מבנה קורס / פעילויות | רשימת פעילויות, סוגים, מועדים, קבוצות |
| LTI / External Tool | זיהוי המורה והמרחב בכניסה לכלי |
| Web Services API | שליפת נתונים אוטומטית — דורש token מאומת + הרשאות |

### 3. החלטת מוצר מחייבת

Teacher Hub הוא **שכבת־על חכמה למורה מעל Moodle**, לא תחליף.  
הוא לא מחזיק תלמידים משלו, לא מנפיק ציונים משלו, לא מחליף את יומן הציונים.  
הוא **מרכז למורה במקום אחד** נתונים אמיתיים מתוך המקורות בטבלה שלעיל: משתתפים, ציונים, דוחות, לוגים, השלמות, פעילות וזמן תרגול — כפי שהם קיימים באותו רגע במרחב.

### 4. יכולת מרכזית: מדידת זמן תרגול

השאלה שהמורה צריך לענות עליה במהירות היא:

> "כמה זמן תלמיד X תרגל ב־24 השעות / 7 הימים / 30 הימים האחרונים, או בטווח תאריכים שאני בוחר?"

- המקור העיקרי הוא **יומני מעקב אמיתיים** (`imported_log_events`) מסונכרנים מ-Moodle Web Services (`core_report_get_log_data`).
- אסור להציג נתוני זמן תרגול על בסיס דמו, הערכה או השלמת פעילות בלבד.
- אם אין לוגים אמיתיים למרחב — יש להציג מצב BLOCKED מפורש בעברית ולא מספרים.

### 5. השלכות למדריך החכם העתידי (Guide Presentation)

מבנה הפרקים המחייב, לפי סדר שמורה חדש היה לומד:

1. כניסה למרחב Moodle
2. היכרות עם מסך המרחב
3. משתתפים
4. ציונים
5. דוחות

לכל פרק יהיו בעתיד צילומי מסך אמיתיים מהמרחב שנבדק, ללא פרטי תלמידים גלויים (טשטוש חובה על שמות/ת"ז/מייל/ציונים אישיים).

### 6. השלכות לקוד הנוכחי — TODOs מעשיים בלבד

מיושם היום (אמת שאושרה בקוד):

- סנכרון משתתפים אמיתי (`core_enrol_get_enrolled_users`) → `imported_students`.
- סנכרון ציונים אמיתי (`gradereport_user_get_grade_items`) → `imported_grades` + `imported_grade_items`.
- סנכרון לוגים אמיתי (`core_report_get_log_data`) → `imported_log_events` (delete-then-insert לפי site+course).
- סנכרון השלמות אמיתי (`core_completion_get_activities_completion_status`) → `imported_task_completion`.
- Practice Time מבוסס לוגים אמיתיים עם מסננים 24h/7d/30d/מותאם + BLOCKED state.
- בידוד site+course בכל hook של ייבוא/שאילתה.
- מצב סנכרון חלקי (partial) עם `logsPartiallyFailed` / `completionsPartiallyFailed`.

חלקי (Roadmap):

- **דוח ניהול למידה** — התוסף הייחודי של משהח (KPIs + 9 גרפים) לא נשלף היום. אין לו endpoint רשמי ב-Web Services; דורש בירור אם קיים API של משהח או שנעבור דרך ייצוא CSV.
- **סמל שאלון / דוח בגרות** — לא זמין ב-Web Services הסטנדרטי. מסומן כ-Roadmap.
- **תזכורות אוטומטיות במייל** — קיים כתוסף במשהח. אנחנו לא מפעילים אותו; רק מציגים למורה אם הוא הוגדר.
- **פעילות בזמן־אמת** — אין תמיכה בסטרימינג ב-WS; אפשר לרענן לוגים ב-polling אם המורה מבקש.

BLOCKED (עד שיש token או הרשאה):

- Web Services שאינם חשופים למורה בלי שיתוף פעולה של מנהל המערכת של משהח.
- קריאה ישירה למסך "ניהול למידה" ללא CSV export ידני מהמורה.

אסור להציג ככפתור עובד:

- כל דוח שאין לו מקור נתונים אמיתי במסד — יוצג כמסך "עוד לא חובר לנתונים אמיתיים" בעברית ברורה, לא כפתור דמו.
