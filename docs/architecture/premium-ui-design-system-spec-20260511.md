## Button states

Each button needs visual states:

- ready
- partial
- missing data
- blocked
- loading
- planned

No active-looking fake buttons.

## Color language

Suggested premium palette:

- deep blue / indigo for primary actions,
- turquoise/emerald for verified data,
- amber for missing reports,
- rose/red for blocked permissions,
- soft slate for neutral areas,
- white/light glass panels for premium clarity.

## Animation rules

Use animations only when they help clarity:

- page entrance,
- card hover,
- button press,
- sync loading,
- status transition.

Do not create distracting animations.

## Screen upgrades

Upgrade existing screens, do not replace blindly:

- Dashboard -> premium command center.
- Tasks -> premium chapters/tasks.
- Students -> participants hub.
- Grades -> grade intelligence.
- Activity -> practice time hub.
- Reports -> report center.
- Export -> export center.
- Import -> smart fallback import wizard.
