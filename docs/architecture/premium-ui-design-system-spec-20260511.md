# Premium UI Design System Spec — Moodle Teacher Hub

## Goal

Create a premium Hebrew RTL teacher interface that feels fast, modern, clear, and high-value.

## Design direction

The UI should use:

- large premium buttons,
- 3D button feeling,
- elegant gradients,
- soft deep shadows,
- rounded cards,
- clean Hebrew typography,
- smooth micro-animations,
- strong empty states,
- clear status badges,
- desktop and mobile support.

## Required main buttons

Main dashboard buttons:

- `סנכרן מרחב`
- `משימות`
- `משתתפים`
- `ציונים`
- `זמנים`
- `דוחות`
- `ייצוא`
- `מה חסר?`

## Button states

Each button needs visual states:

- ready
- partial
- missing data
- blocked
- loading
- planned

No active-looking fake buttons.

## Color language

Suggested premium palette:

- deep blue / indigo for primary actions,
- turquoise/emerald for verified data,
- amber for missing reports,
- rose/red for blocked permissions,
- soft slate for neutral areas,
- white/light glass panels for premium clarity.

## Animation rules

Use animations only when they help clarity:

- page entrance,
- card hover,
- button press,
- sync loading,
- status transition.

Do not create distracting animations.

## Screen upgrades

Upgrade existing screens, do not replace blindly:

- Dashboard -> premium command center.
- Tasks -> premium chapters/tasks.
- Students -> participants hub.
- Grades -> grade intelligence.
- Activity -> practice time hub.
- Reports -> report center.
- Export -> export center.
- Import -> smart fallback import wizard.
