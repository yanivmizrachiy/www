# רשימת בדיקה למנהל Moodle — הפעלת אוטומציה רשמית

מטרת המסמך היא לברר אילו מסלולי אינטגרציה רשמיים זמינים עבור Moodle Teacher Hub, בהרשאות קריאה בלבד, כך שמורים לא יצטרכו להוריד דוחות ידניים.

## מצב נוכחי

- LTI launch חי עובד.
- הקשר קורס מזוהה מתוך Moodle.
- NRPS לא נשלח כרגע.
- AGS לא נשלח כרגע.
- Moodle Web Services לא מאומתים כרגע.
- Teacher Release נשאר NO.

## שאלות LTI / LTI Advantage

1. האם הכלי מוגדר כ־LTI 1.1 או LTI 1.3?
2. אם הכלי מוגדר כ־LTI 1.3, האם LTI Advantage מופעל?
3. האם NRPS מופעל עבור הכלי?
4. אילו שדות פרטיות מותר לשתף דרך NRPS: שמות, אימיילים, תפקידים, מזהים פנימיים?
5. האם AGS מופעל עבור הכלי?
6. אילו הרשאות שירות מאושרות עבור AGS?
7. האם השירותים מוגבלים להקשר הקורס שממנו המורה פתח את הכלי?

## שאלות Moodle Web Services

1. האם Moodle Web Services מופעלים במערכת?
2. האם REST protocol מופעל?
3. האם ניתן ליצור External Service ייעודי לכלי?
4. האם ניתן ליצור חשבון שירות בהרשאות read-only בלבד?
5. האם אפשר לאשר בדיקת חיים ראשונית באמצעות `core_webservice_get_site_info`?
6. האם אפשר לאשר בהמשך פונקציות קריאה לפי צורך:
   - `core_course_get_contents`
   - `core_enrol_get_enrolled_users`
   - `core_completion_get_activities_completion_status`
   - `gradereport_user_get_grade_items`
   - `core_grades_get_grades`
   - `report_log_get_events`

## כללי בטיחות

- אין צורך בהרשאות כתיבה ל־Moodle בשלב זה.
- אין לשמור פרטי התחברות בקוד או ב־GitHub.
- אין לחשוף ערכים רגישים בממשק.
- אין לעקוף הרשאות Moodle.
- אין להשתמש בסקרייפינג של משתמשים או התחברות.

## יעד רצוי

מורה פותח את הכלי מתוך Moodle, המערכת מזהה את הקורס וההקשר, ואז בודקת בצורה בטוחה אילו שירותים רשמיים זמינים למשיכת מידע מתוך אותו הקשר בלבד.
