# System Rules — www / Moodle Teacher Hub

מסמך זה מפרט את כללי העבודה המעשיים לריפו `yanivmizrachiy/www`.

`PROJECT_RULES.md` הוא מקור האמת העליון. המסמך הזה מתרגם את הכללים לעבודה יומיומית בקוד, בתיעוד ובבדיקות.

---

## 1. אמת ותצוגה

- אין דמו.
- אין נתונים מומצאים.
- אין כפתורים שלא מבצעים פעולה אמיתית.
- אין טקסט שמציג API חי בלי הוכחה.
- אין להחליף ערכים חסרים ב־0.
- אין להציג חיבור Moodle אם אין בדיקת API/SSO/LTI אמיתית.

כאשר נתון חסר, משתמשים בניסוח ברור:

- `חסר מהנתונים שיובאו`
- `לא זמין בקובץ הזה`
- `נדרש דוח נוסף ממודל`
- `לא ניתן לחשב ללא לוגים`
- `עדיין לא יובאו נתונים אמיתיים מדוח Moodle`

---

## 2. מקורות נתונים מותרים

מקורות מותרים בלבד:

1. Moodle API / Web Services עם token אמיתי ומאומת.
2. LTI launch אמיתי לצורך כניסה והקשר.
3. ייבוא ידני של דוחות Moodle אמיתיים.
4. קבצי CSV/XLSX/TSV/TXT אמיתיים שהופקו מ־Moodle.
5. טבלאות שהועתקו מתוך Moodle והודבקו למערכת.

אסור להשתמש בנתוני placeholder כתחליף לנתונים אמיתיים.

---

## 3. כללי מכשירים וסשנים

המערכת צריכה לעבוד ממחשב חדר מחשב, מחשב אישי וטלפון נייד.

חובה לשמור על הפרדה בין:

- session
- device
- teacher context
- course context
- imported data source

אין להעביר state בין מכשירים בלי הקשר מאומת.

## 7. כללי ייצוא

סטטוסים:

- CSV — מותר להציג כקיים רק אם הקוד קיים ונבדק.
- Excel/XLSX — מותר להציג כקיים רק אם נוצר קובץ xlsx אמיתי.
- PDF — מותר להציג כקיים רק אם יש יצוא PDF אמיתי או הדפסת דפדפן שמוצהרת כך.
- Print — מותר אם הדפסה עובדת בפועל.

אין לערבב בין PDF אמיתי לבין הדפסה/Save as PDF בלי לציין זאת.
