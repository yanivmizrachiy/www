# Moodle Teacher Hub — Final Product Rules and Roadmap V1

**עודכן:** 2026-05-25
**סטטוס:** מקור-אמת מוצרי משלים ל-`PROJECT_RULES.md` (לא מחליף אותו).
**Teacher Release:** NO (ללא שינוי).

> אם יש סתירה בין מסמך זה ל-`PROJECT_RULES.md`, `PROJECT_RULES.md` קובע.
> מסמך זה מרכז את חזון המוצר ומפת הדרכים בצורה תמציתית וברורה.

---

## 1. החזון הסופי

מורה פותח את Moodle Teacher Hub **מתוך מרחב ה-Moodle שלו** (דרך LTI).
האפליקציה:

1. מזהה את המורה ואת מרחב הלמידה.
2. שואבת **אוטומטית** את מקסימום הנתונים האמיתיים מ-Moodle.
3. **רק אם Moodle חוסם אוטומציה** — משתמשת בייבוא דוחות ידני שכבר עובד, כ-fallback.
4. מציגה דשבורד ועמודים פרימיום בעברית RTL עם נתונים אמיתיים, כפתורים מהירים, וללא תוכן דמו.

המטרה הסופית **אינה** אפליקציית העלאה ידנית. הייבוא הידני הוא רשת ביטחון, לא היעד.

---

## 2. כללי הליבה (מחייבים)

- **המטרה הסופית = שאיבה אוטומטית מקסימלית** מ-Moodle לכל מורה.
- **ייבוא ידני = fallback בלבד** — קיים, עובד, ונשמר. לא מוצג כיעד הסופי.
- **אין דמו, אין נתונים מזויפים** — לא תלמידים, ציונים, לוגים, משימות, זמן תרגול, או מצב "מחובר" מזויפים.
- **אין המצאת יכולות Moodle** — מה שלא אומת חי, לא מוצג כחי.
- **מינימום עבודה ידנית ליניב / מקסימום עבודה בטוחה ואמיתית של ה-AI.**
- **לא שוברים פיצ'רים עובדים** — במיוחד ייבוא, LTI, persistence.
- **PR אחד בכל פעם** — קטן, נקי, עם בדיקות מלאות.
- **אין SQL / deploy / merge ללא אישור מפורש.**
- **Teacher Release נשאר NO** עד שכל שערי הבידוד, האימות החי, וה-persistence עוברים.

---

## 3. תצוגת נתונים — הפרדת אמת

האפליקציה תמיד מבדילה בין:

- נתונים שקיימים באפליקציה
- נתונים שיובאו ידנית מדוח Moodle אמיתי
- נתונים שנגזרו מדוחות שיובאו
- מוכנות קוד/audit
- ראיה חיה מאומתת מ-Moodle/API
- יכולות חסומות

`audit evidence` אינו `live evidence`. אסור לסמן יכולת כ"חיה" ללא רשומת ראיה אמיתית.

---

## 4. פרטיות ובטיחות

- לעולם לא לבקש סיסמת Moodle.
- לעולם לא לגרד credentials, לא להדפיס/לשמור tokens/secrets, לא להעלות `.env`.
- לא לחשוף תגובות Moodle גולמיות עם PII/secrets.
- לא לשמור מידע תלמידים בריפו.
- לא לשנות deploy/secrets/environment ללא בקשה מפורשת.
- בידוד מוחלט בין מורים, מרחבים ומכשירים — אין ערבוב נתונים.

---

## 5. מצב היכולות הנוכחי (אמת)

- **אוטומטי חי:** זהות קורס, זהות מורה (מ-LTI launch).
- **חצי-אוטומטי (ייבוא דוח אמיתי כ-fallback):** משתתפים, ציונים, לוגים, מבנה קורס, השלמת פעילויות, תפקידי מורים.
- **חסום:** Moodle Web Services (אין token מאומת), NRPS, AGS (אין claim חי), זמן תרגול (REFUSE — אין מקור משך), Teacher Release.

**המנוף הגדול ביותר לאוטומציה מלאה:** `MOODLE_WS_TOKEN` אמיתי ממנהל Moodle של משרד החינוך, שמוגדר ב-Render בלבד (לא בריפו). מנוע השאיבה כבר בנוי בקוד — חסר רק ה-token.

---

## 6. מפת דרכים — 8 PRים עד החזון

הסדר נקבע לפי עיקרון: **קודם ערך ישיר למורה (UI/מוצר), אחר כך הרחבת אוטומציה ושחרור.**

1. **Participants Teacher Roles + Privacy-Safe List** — רשימת תלמידים נקייה (שם + פרופיל), ספירת מורים/תלמידים, תפקידים אם יש מקור אמיתי, מזהים רגישים מוסתרים מהרשימה הפשוטה.
2. **Task Type Visual System** — סיווג סוגי משימה אמיתי לאייקון/צבע/כרטיס. מטלה מתוקוונת/בוחן מקוון = ורוד (כמו האייקון הוורוד ב-Moodle). לא ידוע = ניטרלי וכן.
3. **Chapters → Tasks Premium Flow** — פרקים אמיתיים → לחיצה → משימות אמיתיות בפרק, ספירה לכל פרק, השלמה רק אם אמיתית, empty-state ברור.
4. **Practice Time Truth UI** — הפרדת ראיות-לוג (אירועים, ימים פעילים, ראשון/אחרון) מזמן תרגול. זמן תרגול מוצג רק עם מקור משך מאומת, אחרת הודעה כנה בעברית.
5. **Premium Dashboard Teacher Counts** — דשבורד-בית Action Hub עם שם מורה, מרחב, ספירות אמיתיות, סטטוס חיבור תמציתי, עדכון אחרון ב-D/M/YY.
6. **Reports Export Filters** — דוחות אמיתיים עם סינון (תלמיד/משימה/פרק/תאריך) + ייצוא Excel.
7. **Live Moodle Automation Expansion** — רק אחרי שהיסודות יציבים. שליפת NRPS/AGS/WS אמיתית עם evidence חי, דרך ה-Source Router + Truth Engine + Governance.
8. **Multi-Teacher Release Hardening** — הוכחת בידוד 2 מורים/2 קורסים, RLS/בידוד שרת מאומת, פתיחת שער Teacher Release רק עם evidence מלא.

## 8. עקרון העבודה

PR אחד = Work Order אחד. בנייה ב-sandbox → בדיקות מלאות → patch → יניב מחיל בפקודה אחת → Fresh Context Review → merge רק אחרי אישור. שום merge/deploy/SQL ללא אישור מפורש של יניב.
