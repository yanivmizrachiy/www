# דף כללים — Moodle Teacher Hub

עודכן לאחרונה: 2026-04-22

## מטרת המערכת
לבנות מערכת אמיתית למורים שמתחברת למרחב ה־Moodle של משרד החינוך ומציגה את נתוני המרחב האמיתיים של כל מורה, בתצוגה טובה, מהירה, ברורה ונוחה יותר.

## מה המערכת צריכה לעשות
- כניסה מתוך Moodle (LTI) ללא סיסמה נוספת.
- הצגת נתונים אמיתיים בלבד לפי הרשאות המורה והמרחב שלו.
- מסך ראשי ברור בעברית (RTL) עם ניווט מהיר.
- הצגת תלמידים, משימות, פרקים, ציונים, פעילות ודוחות.
- סינון לפי תלמיד/כמה תלמידים/קבוצה/טווח תאריכים.
- תצוגת זמן תרגול מצטבר יומי לכל תלמיד כאשר המידע קיים.
- דוחות נוחים, חזקים וברורים עם אפשרויות יצוא (CSV/XLSX/PDF/הדפסה) לפי מה שנתמך בפועל.
- פעולות עריכה אמיתיות במרחב רק אם Moodle מרשה ולפי הרשאות אמיתיות.

## מה כבר נבנה
- שרת Node.js + Express פעיל.
- מסלול LTI 1.1: `/lti/launch-1p1`.
- מסך Dashboard בעברית: `src/ui/dashboard/dashboard.html`.
- API פנימי בסיסי:
  - `/api/bootstrap`
  - `/api/launches`
  - `/api/students`
  - `/api/tasks`
  - `/api/grades`
  - `/api/activity`
  - `/api/settings`
  - `/api/moodle-summary`
  - `/api/moodle-captures`
  - `/api/export/grades.csv`
- בדיקת תקינות בסיסית: `npm run check`.

## מה כבר עובד
- שרת רץ מקומית ומחזיר health.
- ניתן לבצע Launch בסיסי ולשמור captures מקומיים.
- קיים Dashboard RTL מחובר לנתונים מקומיים/בסיסיים.
- יצוא CSV בסיסי זמין לציונים.

## מה עדיין חסר
- חיבור מלא ויציב ל־Moodle Web Services בנתוני אמת לכל הישויות.
- השלמת זמן פעילות יומי ברמת דיוק מקסימלית לפי נתונים זמינים.
- דוחות מתקדמים משולבים (ציונים/משימות/זמנים/פעילות) עם פילטרים מלאים.
- יצוא XLSX, PDF והדפסה מלאה לכל סוגי הדוחות.
- יכולות עריכה מלאות (לפי הרשאות אמיתיות) מתוך הממשק.
- הקשחה לפריסה יציבה ב־HTTPS עבור שימוש רוחבי למורים.

## כללים קבועים (ברזל)
- אין דמו.
- אין נתונים מומצאים.
- אין כפתורים/יכולות שלא מבצעים פעולה אמיתית.
- אין מסכים "יפים" ללא חיבור אמיתי לנתונים.
- אין כפילויות קוד/מסמכים/זרימות.
- אין קבצים מיותרים או רכיבים ללא ערך מוצרי.
- שמירה על שמות קבצים ברורים ומבנה תיקיות מסודר.
- כל פיתוח חדש חייב להישען על אמת נתונים והרשאות Moodle.

## מה אסור
- להציג מידע שלא הגיע ממקור אמת.
- להוסיף פונקציונליות שאינה פועלת מקצה לקצה.
- לעקוף הרשאות של Moodle.
- להכניס שכפולים במקום שימוש ברכיב/שירות קיים.
- להשאיר "קוד עתידי" לא פעיל ללא שימוש אמיתי.

## מה מתוכנן להמשך
- אינטגרציית Moodle מלאה לסטודנטים/משימות/ציונים/אירועים.
- תצוגת פרקים ונושאים עם ניווט חכם וצבעוניות עקבית.
- הרחבת דוחות ויכולות פילטור לפי כיתה/קבוצה/זמנים.
- הרחבת יצוא לפורמטים נוספים בהתאם לנתונים הקיימים בפועל.
- שיפור UX בעברית עם דגש על מהירות ובהירות.
