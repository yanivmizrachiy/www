# בדיקת כפילויות — מדריך /guide

מקור אמת אחד לכפתורים: `src/data/guideButtons.ts`. כל כפתור מוסבר **פעם אחת**; כפתור שחוזר בכמה מקומות מקבל הערת "ראו גם / מופיע גם" (`also`) במקום הסבר חוזר.

## כפתורים שמופיעים בכמה מקומות (הוסבר פעם אחת + "ראו גם")
- **ציונים** — מוסבר בסרגל העליון; מצוין ש"מופיע גם בתפריט המשתמש וגם בתפריט הקורס". בתפריט המשתמש ובתפריט הקורס יש הערת הפניה, לא הסבר חוזר.
- **דוחות** — הופרד: דוחות **אישיים** (תפריט המשתמש) מול דוחות **הקורס** (תפריט הקורס) — יעדים שונים, לכן שני פריטים עם הבהרה, לא כפילות.
- **החלפת תפקיד ל...** — מוסבר בתפריט המשתמש; מצוין ש"מופיע גם באפשרויות נוספות של הקורס".
- **משתתפים / ציונים** בתפריט הקורס — הפניה קצרה לאזור המלא ("ראו אזור"), לא הסבר כפול.
- **כפתור +** (מצב עריכה) — מפנה לאזור "הוספת משאב או פעילות", לא מסביר שוב את הבורר.
- **הורדת טבלת משתתפים** — מפנה למסלול "הורדת דוחות וקבצים".

## פעולות דומות שאוחדו לפריט אחד
- סינון משתתפים (מילים / מצב / תפקיד / שיוך / אי־פעילות) → פריט אחד: "מצמצמים את הרשימה לפי מה שבחרתם".
- פורמטי יצוא ציונים (Excel / ODS / CSV/Text / XML) → מוצגים כ"בחירת פורמט" תחת כפתור **יצוא**, לא ארבעה כרטיסים.

## מסלול "הורדת דוחות וקבצים"
אינו משכפל הסברים — הוא מפנה (jump) אל הכפתורים באזורים ציונים/דוחות/משתתפים.

## בדיקה
נבדק שאין שני כרטיסים עם אותו שילוב של: label + area + action. כפתורים חוזרים מסומנים ב-`also` בלבד.
