## חלק ב׳ — Moodle Teacher Hub / WWW

זהו הכלי הקיים בריפו `www`.  
מורה מוסיף אותו כקישור חיצוני / External Tool במרחב Moodle שלו.  
הקישור המרכזי הוא אחיד, והמערכת מזהה את המורה/קורס/מרחב לפי Moodle context.

עקרונות:

- אפליקציה מרכזית אחת.
- אין deployment נפרד לכל מורה.
- אין קוד נפרד לכל מורה.
- כל מורה רואה רק את המרחב שלו.
- כל קורס מופרד מקורס אחר.
- כל שיפור ב־`www` מתעדכן לכל המורים דרך אותו runtime.
- נתונים אמיתיים בלבד.
- אין דמו.
- אין נתונים מומצאים.
- Teacher Release נשאר **NO** עד שכל שערי האימות עוברים.

## אוטומציה מקסימלית לשליפת נתונים ב־WWW

בכל תחום שבו עובדים על Moodle Teacher Hub / WWW, המטרה היא אוטומציה מקסימלית במידת האפשר.

סדר עדיפות לשליפה:

1. LTI launch context.
2. LTI 1.3 claims.
3. NRPS — רשימת משתתפים, אם claims והרשאות קיימים.
4. AGS — ציונים/משימות, אם claims והרשאות קיימים.
5. Moodle Web Services API עם token אמיתי ומאומת.
6. קישורי ייצוא דוחות מתוך Moodle.
7. ייבוא דוחות אמיתיים: Participants, Gradebook, Logs, Activity Completion / Course Structure.
8. ניתוח קבצים שהמורה מעלה.
9. התאמת נתונים בין מקורות.

חובה:

- למקסם אוטומציה.
- לזהות אוטומטית מה זמין.
- להציג למורה מה חסר.
- להציע את הפעולה הבאה הכי פשוטה.
- לא להמציא נתונים.
- לא להציג אוטומציה מזויפת.
